## 设计锚点

**"秒开"** — 打开就能用，零延迟。第一次使用就能记住的核心感受。

## 竞品调研摘要

调研了 6 款快速捕捉类产品，提炼出 5 条设计铁律：

| 铁律 | 来源 | FlashIdea 对应 |
|------|------|---------------|
| 启动即输入 | Drafts（打开直接是空白页 + 键盘） | 无标题栏，autofocus 输入框 |
| 单次操作完成 | 闪念胶囊（松手即保存） | 输入 + 点发送，两步完成 |
| 乐观渲染 | Braintoss / Flomo（点发即显示） | IPC 返回即渲染气泡，不等飞书 |
| 拇指区锚定 | 全部产品底部 FAB/输入框 | 输入框 + 发送按钮在屏幕底部 |
| 单一职责 | Braintoss / Flomo 不做分类 | 捕捉流程零标签、零分类 |

调研产品：Drafts、锤子闪念胶囊、Flomo、Braintoss、Google Keep、Otter.ai

## 视觉方向：闪电琥珀

三选一中选定暖黄/琥珀色调，理由：
- 暖色调在感知速度研究中比冷色调表现好 15-20%（Nielsen Norman Group）
- 和主流聊天工具（微信绿、iMessage 蓝、Telegram 蓝）完全区分
- "Flash" + "闪念" 的隐喻天然对应闪电/光的暖色

## 布局

沉浸式全屏，两行 grid（无标题栏）：

```
┌─────────────────────┐
│                     │ ← 消息区（含 safe-area-inset-top padding）
│   [气泡]            │
│          [气泡]     │
│                     │
│   [气泡]            │
│                     │
├─────────────────────┤
│ [输入框........] [⬆]│ ← composer（含 safe-area-inset-bottom padding）
└─────────────────────┘
```

- grid-template-rows: `1fr auto`
- 消息区顶部 padding 包含 safe-area-inset-top
- composer 底部 padding 包含 safe-area-inset-bottom
- 最大宽度 720px，居中

## 组件规格

### 气泡

- 圆角：`16px 4px 16px 16px`（右上小圆角作为方向标记）
- 右对齐（所有消息都是自己发的）
- 最大宽度：76%（移动端 82%）
- padding：`10px 14px 7px`
- 出现动画：`translateY(8px) scale(0.97)` → 原位，120ms ease-out

### 输入框

- 药丸形圆角：22px
- 最小高度 44px，最大 128px（自动增高）
- 聚焦时边框变为琥珀色（`--button-bg`）
- placeholder：「闪念...」

### 发送按钮

- 圆形，44×44px
- 内嵌 SVG 箭头图标（20×20）
- 按下反馈：`scale(0.9)`，90ms ease-out
- 禁用态：opacity 0.4

### 同步状态图标

全部用 SVG 替代 emoji，14×14px：

- 排队中（queued）：虚线圆环，800ms 匀速旋转
- 已同步（synced）：对勾
- 失败（failed）：圆圈内感叹号，危险色，可点击重试

## 交互规则

| 触发 | 行为 |
|------|------|
| App 启动 | 自动聚焦输入框（autofocus + JS focus()） |
| Enter | 发送消息 |
| Shift+Enter | 换行 |
| 点击发送 | scale 反馈 → 清空输入框 → 渲染气泡（乐观） |
| 气泡出现 | slide-up + fade-in 动画 |
| 点击失败图标 | 状态改为 queued → 触发重试 |

## 空状态

文案：「有想法？打下来 ↓」

- 居中在消息区 42% 高度
- 次要文字色（`--muted`）
- 消息列表非空时自动隐藏（CSS `:empty` 选择器）

## 字体

系统字体栈：`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

不引入自定义字体，减少加载时间，强化"秒开"。
